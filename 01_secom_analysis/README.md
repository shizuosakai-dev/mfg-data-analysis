# 半導体ウェハ製造データによる不良品予測（SECOM Dataset）

半導体製造プロセスにおけるセンサーデータから、ウェハのPass/Fail（良品・不良品）を予測する機械学習プロジェクト。

## 背景・目的

半導体製造プロセスでは、数百〜数千のセンサーがウェハの状態をリアルタイムで監視している。本プロジェクトでは、これらのセンサーデータからウェハの不良を予測できるか、また予測に有効なセンサーを絞り込めるかを検証した。

製造プロセスの品質管理には大きく2つのアプローチがある。

- **Feed-Forward（FF）制御**：上流工程の測定結果をもとに、同一ロットの下流工程条件を補正する
- **Feedback（FB）制御**：最終検査結果をもとに、後続ロットの上流工程条件にフィードバックする

本プロジェクトはFB的な位置づけであり、「不良を見逃さないこと（Fail recallの最大化）」を最優先の評価方針とした。不良ロットを良品と誤判定（偽陰性）した場合、後工程・出荷後まで不良が流出するリスクに加え、装置起因の不良である場合には同一装置で処理される後続ロットにも同一不良が連続発生するリスクがある。これは誤警報（偽陽性）による追加検査コストよりもインパクトが大きいと考え、Fail recallを優先する方針とした。

## データセット

[UCI Machine Learning Repository - SECOM Dataset](https://archive.ics.uci.edu/dataset/179/secom)

- サンプル数：1,567（ウェハロット）
- 特徴量数：590（センサー計測値）→ 前処理後 446（欠損率50%以上の列を除去して562列、さらに分散ゼロの定数列116列を除去）
- Pass/Fail比率：Pass 93.4%（1,463件） / Fail 6.6%（104件）、約12:1のクラス不均衡
- ラベル：Pass（-1） / Fail（1）

## 分析フロー

1. **EDA**
   - Pass/Fail分布の確認
   - 欠損値ヒートマップによる欠損パターンの可視化（欠損率上位30列、最大約91%欠損の列も存在）
   - センサー分布のヒストグラム確認
   - 欠損率50%以上の列を除去（590列 → 562列）
   - 分散ゼロの定数列を除去（116列除去、最終446センサー特徴量）
   - Pass/Fail間の相対平均差分メトリクスによるセンサー選別（独自指標）
   - 欠損値がPass/Fail間で構造的に均一であることを確認（欠損自体は不良シグナルではないと判断）

2. **前処理・モデリング**
   - train/testスプリット（80:20、stratify）
   - 欠損値補完（中央値）、標準化
   - ロジスティック回帰（baseline / class_weight='balanced'）
   - Random Forest
   - XGBoost
   - SVM（C, gammaのパラメータ調整）

3. **クラス不均衡への対応**
   - SMOTE（Synthetic Minority Over-sampling Technique）による少数クラスの合成サンプル生成
   - 閾値調整によるrecall/precisionトレードオフの検証
   - Precision-Recall曲線によるモデル比較（ロジスティック回帰 vs SVM）

4. **特徴量選択実験**
   - 相対平均差分の大きい上位Nセンサーのみを使用したロジスティック回帰の性能比較（N=5〜446）

## 主な結果

### 特徴量数（N）別のロジスティック回帰性能

| 上位センサー数N | Fail Recall | Fail Precision |
|---|---|---|
| 5 | 0.52 | 0.07 |
| 10 | 0.29 | 0.05 |
| 20 | 0.38 | 0.06 |
| **50** | **0.67** | **0.14** |
| 100 | 0.43 | 0.12 |
| 446（全列） | 0.19 | 0.11 |

全センサーを使うより、Pass/Fail間の相対平均差分が大きい上位50センサーに絞ったほうがFail recallが大幅に改善する（N=446で0.19 → N=50で0.67）。

### モデル間比較（最終評価、N=50前提のモデルを含む）

| モデル | Fail Recall | Fail Precision |
|---|---|---|
| ロジスティック回帰（N=50, balanced） | 0.67 | 0.14 |
| ロジスティック回帰（N=50, balanced, 閾値0.1） | 1.00 | 0.07 |
| SVM（初期パラメータ, 上位20） | 0.38 | 0.07 |
| SVM（C=1, gamma=0.01, 上位20） | 0.67 | 0.08 |
| ロジスティック回帰 + SMOTE（N=50） | 0.57 | 0.13 |
| ロジスティック回帰（全446列） | 0.19 | 0.11 |
| Random Forest（全446列） | 0.00（訓練データでは1.00） | 0.00 |
| Random Forest（N=50） | 0.05 | 0.33 |
| XGBoost（全446列） | 0.00 | 0.00 |

### 考察

- **特徴量選択の効果が最大**：ノイズの多いセンサーを除外し、Pass/Fail差分の大きい上位50列に絞ることが最も性能改善に寄与した
- **木系モデル（Random Forest, XGBoost）は軒並み不良判定不能**：Random Forestは訓練データに対しては完全一致（recall/precision共に1.00）する一方、テストでは不良を1件も検出できず、著しい過学習を示した。SECOMデータは複雑な非線形パターンよりも弱い線形シグナルに近い可能性が高い
- **SMOTEは今回のデータでは効果なし**：Fail（不良）サンプルがセンサー空間上で散在しており、k-NN補間による合成サンプルが有効な決定境界の学習につながらなかった（SMOTE適用前 recall 0.67 → 適用後 0.57）
- **閾値調整はrecall/precisionのトレードオフを可視化する手段の一つ**：閾値を0.1まで下げるとrecall 1.00を達成できるが、precisionは0.07まで低下する。この水準は「一次フィルター」としても実用的とは言い難い。テストデータ（Pass 293件・Fail 21件）で計算すると、フラグが立つのは合計約300件（全体の95.5%）にのぼり、Pass 293件中約279件を誤って要検査に分類することになる。これはほぼ全数検査と変わらず、検査工数削減という「フィルター」本来の目的を果たさない。運用・保守コストに対するリターンを考えると、この閾値設定は現実的ではない
- **センサーデータ自体の予測力には限界がある**：ロットばらつき・ライン差・確率的プロセス要因など、データに含まれない要因の影響も示唆される（今後の課題）

## 使用技術

Python / pandas / numpy / matplotlib / seaborn / scikit-learn / XGBoost / imbalanced-learn（SMOTE）

## 学び・今後の課題

- 製造データにおける評価指標選定は、業務上のコスト構造（見逃しコスト vs 誤検知コスト）に基づいて行う必要がある
- 特徴量選択がモデル性能に与える影響は大きく、全変数投入が必ずしも最適ではない
- センサーデータのみでは説明しきれない不良要因（ロット間ばらつき等）が存在する可能性があり、今後は工程パラメータや時系列情報の追加検討が課題
